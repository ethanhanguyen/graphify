# PR 3.1 — Caching + Agent Integration

**Epoch:** 3 (Integration & Scale)
**Depends on:** PR 2.1 (Call Resolution), PR 2.3 (Hybrid Search)
**Estimated effort:** 2 weeks
**Impact:** High for agent UX

## Context

With call resolution (PR 2.1) and hybrid search (PR 2.3), Graphify can answer complex code questions. This PR makes those answers fast (query caching) and injects them directly into agent workflows (skills, hooks, context injection).

This merges P5 (Caching) with P12 (Agent Integration) because query cache results feed directly into PreToolUse hook enrichment, and PostToolUse hooks drive cache invalidation.

## Deliverables

### 1. Query Result Cache
- New file: `graphify/query_cache.py`
  - `QueryCache` class
  - `cache_key(query_text, mode, depth, edge_types, min_confidence) → str` — SHA256 hex digest
  - `get(cache_key) → Optional[cached_result, timestamp]`
  - `set(cache_key, result, ttl_seconds=300)` — writes to `graphify-out/query_cache/{prefix}/{key}.json`
  - `invalidate(node_ids)` — targeted cache eviction: only evict cached results that reference any of the given nodes
  - `invalidate_file(file_path)` — evicts cached results referencing nodes in a specific file
  - `invalidate_all()` — full cache flush (only on `graphify build`)
  - `stats() → {hits, misses, size_mb, entry_count}`
  - Configurable TTL per query type: `context=300s`, `impact=600s`, `query=120s`, `trace=600s`
  - Auto-expiry: stale entries evicted on next access (no background thread needed)

### 2. Cache Integration in MCP Server
- Modified: `graphify/serve.py`
  - `_cached_query(tool_name, params, query_fn)` — cache wrapper
  - Before running query, check `G.graph["query_cache"].get(key)`
  - On cache hit, return cached result with `_cached: true` metadata
  - On cache miss, run query, cache result, return with `_cached: false`
  - Cache hash includes graph schema version — auto-invalidates on rebuild

### 3. Agent Skill Templates
- New directory: `graphify/skills/`
- New file: `graphify/skills/__init__.py`
  - `SkillGenerator` class
  - `generate_skills(graph, output_dir) → List[generated_file_paths]`
  - Output to `.claude/skills/graphify/` (namespaced, does not overwrite user skills)
- New file: `graphify/skills/exploring.py`
  - "Exploring" skill: graph navigation via `query`, `context`, `query_graph` tools
  - Template includes: available node types, common query patterns, example usage
  - Dynamic content: god nodes from graph, community structure, top entry points
- New file: `graphify/skills/debugging.py`
  - "Debugging" skill: call chain tracing via `context`, `trace`, `impact` tools
  - Template includes: how to trace a bug through call chains, how to find root cause
  - Dynamic content: error nodes in the graph, test fixtures, logging patterns found
- New file: `graphify/skills/impact.py`
  - "Impact Analysis" skill: blast radius via `impact`, `detect_changes` tools
  - Template includes: pre-change impact assessment workflow, risk level interpretation
  - Dynamic content: common change patterns, high-risk areas identified by community
- New file: `graphify/skills/refactoring.py`
  - "Refactoring" skill: dependency mapping via `impact`, `context` tools
  - Template includes: safe rename workflow, dependency visualization, multi-file coordination
  - Dynamic content: module boundaries, cross-community dependencies, god nodes

### 4. Per-Community SKILL.md Generation
- New file: `graphify/skills/repo_skills.py`
  - `generate_community_skills(graph, communities, output_dir) → List[path]`
  - For each Leiden community with ≥3 files and cohesion > 0.6:
    - Extract: key files, entry points, execution flows, cross-community connections
    - Generate SKILL.md with: community name, purpose summary, key symbols, patterns, test areas
    - Output to `.claude/skills/generated/{community-slug}.md`
  - `get_community_cohesion(community) → float` — % of edges within community vs crossing boundaries
  - `summarize_community(graph, community_nodes) → CommunitySummary` — one-paragraph functional description

### 5. Agent Hooks
- New file: `graphify/skills/hooks.py`
  - `HooksGenerator` class
  - `generate_pre_tool_use_hook(output_dir)` → writes `pre-tool-use-graphify.sh`
    - Intercepts tool calls (search, file reading)
    - Enriches with relevant graph context: if searching for "auth", inject top 5 auth-related symbols from hybrid search
    - Only fires when `graphify-out/` exists
  - `generate_post_tool_use_hook(output_dir)` → writes `post-tool-use-graphify.sh`
    - Detects file writes (git hook trigger)
    - Checks cache staleness: `G.graph["query_cache"].invalidate_file(changed_file)`
    - If ≥5 files changed, prompts: "Consider running `graphify update .` to refresh the knowledge graph"
  - `generate_hooks(output_dir)` → both
  - Output to `.claude/hooks/`

### 6. Enhanced Agent Context Injection
- New: `graphify claude setup` (extends existing `graphify claude install`)
  - `graphify claude setup --harness opencode` → detects harness, writes appropriate config
  - `graphify claude setup --harness cursor` → cursor-specific MCP config
  - `graphify claude setup --harness copilot` → copilot-specific config
  - `graphify claude setup --all` → all harnesses found on system
  - Auto-detection: checks for `~/.config/opencode/config.json`, `~/.cursor/mcp.json`, etc.
- Writes AGENTS.md / CLAUDE.md section:
  - Available tools and usage patterns
  - God nodes summary for quick orientation
  - Community structure overview
  - Key entry points

### 7. CLI Commands
- New subcommand: `graphify skills`
  ```
  graphify skills                    # Generate 4 base skills
  graphify skills --repo             # Generate per-community SKILL.md
  graphify skills --hooks            # Generate PreToolUse + PostToolUse hooks
  graphify skills --all              # All of the above + setup
  ```
- New subcommand: `graphify cache`
  ```
  graphify cache stats               # Show hit/miss/size stats
  graphify cache invalidate --file X # Invalidate cache for specific file
  graphify cache flush               # Full cache flush
  ```
