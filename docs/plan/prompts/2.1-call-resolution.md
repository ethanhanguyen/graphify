# PR 2.1 — Call Resolution Engine

**Epoch:** 2 (Core Intelligence)
**Depends on:** PR 1.2 (Code Schema + Typed Indexing)
**Estimated effort:** 3-4 weeks
**Impact:** CRITICAL — enables "what calls what?", "what breaks if I change X?"

## Context

This is the single highest-impact PR. Currently Graphify cannot answer code-specific questions like "what functions call `validateUser`?" or "what depends on `UserService`?" because it has no call resolution — cross-file call targets are never resolved, imports are not followed, and type-based dispatch (MRO, receiver inference) doesn't exist.

This PR implements a 6-stage call resolution DAG modeled after GitNexus's architecture, plus a `LanguageCallProvider` plugin system for extensibility.

## Deliverables

### 1. Language Call Provider Protocol
- New file: `graphify/language_provider.py`
  - `LanguageCallProvider` abstract base class / protocol
  - Required methods: `import_resolver`, `call_extractor`, `receiver_inferrer`, `mro_strategy`, `import_semantics`
  - `register_provider(language, provider)` — global registry
  - `get_provider(language) → LanguageCallProvider`
  - Languages discovered via entry points: `graphify.language_providers` setuptools group

### 2. Import Resolution
- New file: `graphify/imports.py`
  - `resolve_import(target_name, from_file, all_files, language) → Optional[resolved_file]`
  - 4 resolution strategies per language (delegated to `LanguageCallProvider.import_semantics`):
    - `named` — exact match (Python `from foo import bar`)
    - `wildcard_leaf` — match last segment (* imports)
    - `wildcard_transitive` — match any segment depth
    - `namespace` — directory-based (Go packages, Java packages)
  - `build_import_graph(files) → DiGraph` — dependency graph of imports between files
  - `resolve_all_imports(file, parsed_ast, all_files) → Dict[name, resolved_file]` — resolves all imports in a single file

### 3. Call Extraction
- New directory: `graphify/call_extractors/`
- New file: `graphify/call_extractors/__init__.py`
- New file: `graphify/call_extractors/typescript.py`
  - Extracts: `identifier()`, `obj.method()`, `this.method()`, `super.method()`, `new Class()`, `await fn()`, arrow callbacks
  - Handles: nested member expressions (`a.b.c()`), optional chaining (`obj?.method()`)
- New file: `graphify/call_extractors/python.py`
  - Extracts: `func()`, `obj.method()`, `self.method()`, `super().method()`, `Class()`, decorator calls, `await fn()`
  - Handles: `@decorator` syntax, positional + keyword arguments
- New file: `graphify/call_extractors/go.py`
  - Extracts: `Func()`, `obj.Method()`, `pkg.Func()`, `go func()`, `defer func()`
  - Handles: goroutine launches, interface method dispatch
- New file: `graphify/call_extractors/java.py`
  - Extracts: `method()`, `this.method()`, `super.method()`, `obj.method()`, `new Class()`, static calls, annotation method refs
  - Handles: generics erasure, varargs, lambda → SAM conversion
- Common: `ExtractedCallSite` dataclass — `caller`, `caller_file`, `callee_name`, `callee_receiver`, `arity`, `is_constructor`, `source_location`, `confidence`

### 4. Receiver Inference
- New file: `graphify/receiver.py`
  - `infer_receiver(call_site, enclosing_scope, all_classes) → Optional[resolved_type]`
  - `SelfInferrer` — resolves `self`/`this` by walking up AST to find enclosing class
  - `ConstructorInferrer` — resolves `new X()` and `X()` class constructors
  - `ChainInferrer` — resolves `a.b.c()` by walking the chain: a.type → b.type → c
  - `ImportInferrer` — resolves imported identifiers

### 5. MRO Strategy
- New file: `graphify/mro.py`
  - `MROStrategy` protocol with `resolve(method_name, class_hierarchy) → Optional[(target_class, confidence)]`
  - `C3Linearization` — Python MRO (C3 linearization algorithm)
  - `FirstWins` — Java/C#/C++/TypeScript/Go (first definition wins in linearization order)
  - `RubyMixin` — Ruby-style mixin insertion order
  - `NoneMRO` — single inheritance, no resolution needed
  - `build_class_hierarchy(graph, file_list) → Dict[class_name, List[superclass_names]]` — builds from EXTENDS edges

### 6. Cross-File Type Propagation
- New file: `graphify/cross_file.py`
  - `build_file_dependency_graph(import_map) → DiGraph` — nodes are files, edges are imports
  - `compute_scc_order(dependency_graph) → List[Set[file]]` — Tarjan's SCC algorithm
  - `propagate_types(scc_order, files, all_symbols)` — process each SCC, resolve types, repeat for next SCC
  - Ensures all symbols referenced across files are resolved before call linking

### 7. Call Resolution DAG
- New file: `graphify/call_dag.py`
  - `CallResolutionDAG` — orchestrates the 6 stages:
    1. **Extract** — tree-sitter per-file → `List[ExtractedCallSite]`
    2. **Classify** — tag each call site: direct, method, constructor, import_dispatch, anonymous
    3. **Infer Receiver** — resolve `self`/`this`, constructor types, chained expressions
    4. **Select Dispatch** — pick MRO strategy for the language, find matching method
    5. **Resolve Target** — cross-file resolution: follow imports to find target file → find matching symbol
    6. **Emit Edge** — create `CALLS` edge with confidence score
  - `run_dag(files, language, graph) → List[CallEdge]` — runs all 6 stages
  - Each stage is incremental: only reprocess changed files (SHA1-based detection)

### 8. MCP Tools
- New: `context` tool in `serve.py`
  ```
  context({name: "validateUser"})
    → symbol: {kind, file, line, language}
    → incoming: {calls: [{caller, file, line, confidence}], imports: [...]}
    → outgoing: {calls: [{callee, file, line, confidence}]}
    → processes: [{name, step_index, total_steps}]
  ```
- New: `impact` tool in `serve.py`
  ```
  impact({target: "UserService", direction: "upstream", minConfidence: 0.8, maxDepth: 5})
    → target: {kind, file}
    → upstream: {depth_1: [...], depth_2: [...], ...}  // or "downstream"
    → summary: {total_affected, risk_level: "low"|"medium"|"high"|"critical"}
  ```
  Risk level heuristics: `affected > 50 && max_depth > 3` → critical, `affected > 10` → high, etc.

### 9. Build Integration
- Modified: `graphify/build.py`
  - After typed extraction, run `CallResolutionDAG.run_dag()` per language group
  - Emit `CALLS` edges into graph with confidence scores
  - `G.graph["call_resolution_stats"] = {resolved: 452, unresolved: 23, total: 475}`
