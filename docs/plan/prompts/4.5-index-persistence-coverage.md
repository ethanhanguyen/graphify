# PR 4.5 — Index Persistence + Coverage Remediation

## Problem
1. Indexes rebuilt on every graph load — wasted O(N+E) work
2. `index.py` 0% coverage, `code_emitter.py` 0%, `serve.py` 17%

## Goal
1. Serialize indexes alongside graph.json
2. 95% coverage on index.py and code_emitter.py
3. 60%+ coverage on serve.py

## Changes

### 1. `graphify/index.py` — Serialization
- `CompositeIndex.to_dict()` — serialize to JSON-safe dict
- `CompositeIndex.from_dict(data)` — reconstruct from dict
- `save_indexes(G, path)` — write `<path>.index.json`
- `load_indexes(path)` — read if exists
- `load_or_build_indexes(G, path)` — try load, else build

### 2. `graphify/export.py` — Save indexes after graph export
Call `build_indexes(G)` + `save_indexes(G, output_path)` in `to_json()`.

### 3. `graphify/serve.py` — Load indexes on graph load
Call `load_or_build_indexes(G, graph_path)` in `_load_graph_file()`.

### 4. `tests/test_index.py` (NEW — target 95%)
- EdgeTypeIndex: lookup by type, empty, unknown type
- NodeLabelTrie: prefix search, exact, empty, dedup
- ConfidenceBitmap: tier filter, edge lookup
- CompositeIndex: combined query, serialization round-trip
- save/load functions

### 5. `tests/test_code_emitter.py` (NEW — target 95%)
- Edge emission with all confidence tiers
- Fluent API chaining
- `as_dicts()` output format
- `stats()` counters, `clear()` reset
- `tag_node()` node type mapping

### 6. `tests/test_serve.py` (extend — target 60%+)
- `_bidirectional_shortest_path`, `_weighted_dijkstra`, `_astar`
- `_score_nodes` ranking, `_subgraph_to_text` output
- `_communities_from_graph` reconstruction
