# PR 1.3 — Advanced Traversal

**Epoch:** 1 (Foundation)
**Depends on:** PR 1.2 (Code Schema + Typed Indexing)
**Estimated effort:** 1 week
**Impact:** High — massive query speedup

## Context

Graphify currently uses basic BFS/DFS (depth-limited, hardcoded `depth=3`) and `nx.shortest_path()` for all queries. This PR adds bidirectional BFS, weighted Dijkstra, A* with community heuristic, and cardinality-aware query planning.

Since PR 1.2 provides typed edge indexes and confidence bitmaps, cardinality estimation and edge-type filtering are now O(1) operations, making traversal optimization both fast and accurate.

## Deliverables

### 1. Bidirectional BFS
- Modified: `graphify/serve.py`
  - `_bidirectional_shortest_path(G, src, tgt, max_hops=6, edge_types=None, min_confidence=0.5) → List[path_nodes]`
  - Forward frontier from src, backward frontier from tgt, meet-in-the-middle
  - Uses `G.graph["indexes"]` for O(1) edge-type filtering at each hop
  - For a 6-hop path: from 15.6B candidates (unidirectional) → ~250k (bidirectional)
  - Exposed as `mode="bidirectional"` in `query_graph` and `shortest_path` MCP tools

### 2. Weighted Dijkstra
- Modified: `graphify/serve.py`
  - `_weighted_dijkstra(G, src, tgt, weight_field="confidence", edge_types=None) → List[path_nodes]`
  - Uses edge `confidence` as weight (lower = more reliable path for blast radius)
  - Also supports `weight_field="hops"` (unweighted) and `weight_field="complexity"` (future)
  - Exposed via `shortest_path(weighted=True)` MCP parameter
  - Useful for "cheapest dependency chain" queries where confidence matters

### 3. A* with Community Heuristic
- Modified: `graphify/serve.py`
  - `_astar(G, src, tgt, heuristic="community", edge_types=None)` → List[path_nodes]
  - Heuristic: `h(n) = 1.0 if community(n) != community(target) else 0.5`
  - Encourages traversal to stay within the target's community (functional cohesion)
  - Also supports `heuristic="degree"` (prefer low-degree nodes) and `heuristic="combined"` (default)
  - Exposed as `mode="astar"`

### 4. Query Planning
- Modified: `graphify/serve.py`
  - `_select_best_start_node(candidates, G) → node_id` — picks lowest-degree node from candidate set to minimize fan-out
  - `_prefer_extracted_edges(frontier, G, min_confidence=0.8)` — sorts frontier by edge confidence tier before expanding
  - `_estimate_cardinality(G, edge_type, from_node) → int` — uses `G.graph["indexes"]` for O(1) cardinality lookup
  - For multi-hop queries: re-evaluate start node at each hop with updated degree info

### 5. Backward Compatibility
- All existing `_bfs()` and `_dfs()` functions preserved as-is
- `query_graph` default mode remains `"bfs"` (will change to `"hybrid"` in PR 2.3)
- New modes are explicit opt-in: `mode="bidirectional"`, `mode="astar"`, `mode="dijkstra"`
- `shortest_path` gains `weighted=True` and `mode="bidirectional"` parameters

### 6. Tests
- New file: `tests/test_traversal.py`
  - Test bidirectional BFS on graphs of 100, 1k, 10k nodes, verify correctness vs unidirectional BFS
  - Test weighted Dijkstra: verify it prefers high-confidence paths over low-confidence
  - Test A* community heuristic: verify it finds paths within target community faster
  - Test cardinality-based start node selection: verify it picks lowest-degree node
  - Test edge confidence filtering: verify EXTRACTED paths preferred over INFERRED
  - Performance tests: 6-hop path on 100k-node graph, bidir must be >100x faster than unidir
- Modified: `tests/test_serve.py` — add mode parameter tests for all new traversal modes
