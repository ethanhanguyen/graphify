# PR 1.2 — Code Schema + Typed Indexing

**Epoch:** 1 (Foundation)
**Depends on:** PR 1.1 (Fork + Baseline + Fixtures)
**Estimated effort:** 2-3 weeks
**Impact:** Foundation for all code intelligence phases

## Context

Graphify currently uses generic "concept" nodes and untyped edges for all code. This PR introduces a typed code schema (44 node types, 21 edge types) and builds type-aware indexes that make all subsequent query optimizations and code intelligence features possible.

The indexing layer (originally Phase 2) is merged here because typed edge indexes are 10x more useful than generic edge indexes. Build them together.

## Deliverables

### 1. Code Schema Definition
- New file: `graphify/code_schema.py`
  - Node type enum with 44 types: `Function`, `Class`, `Method`, `Interface`, `Route`, `Tool`, `Process`, `Module`, `Enum`, `TypeAlias`, `Constructor`, `Struct`, `Trait`, `Namespace`, `Variable`, `Parameter`, `Import`, `Export`, `Decorator`, `Property`, `Getter`, `Setter`, `Lambda`, `Closure`, `DatabaseTable`, `DatabaseColumn`, `Middleware`, `Hook`, `Event`, `Listener`, `Config`, `EnvironmentVariable`, `Secret`, `Error`, `Warning`, `Debug`, `Info`, `GenericType`, `UnionType`, `IntersectionType`, `OptionalType`, `LiteralType`, `ArrayType`, `MapType`
  - Edge type enum with 21 types: `CALLS`, `IMPORTS`, `EXPORTS`, `EXTENDS`, `IMPLEMENTS`, `METHOD_OVERRIDES`, `HANDLES_ROUTE`, `STEP_IN_PROCESS`, `MEMBER_OF`, `DECORATED_BY`, `TYPED_AS`, `DEPENDS_ON`, `INJECTS_INTO`, `THROWS`, `CATCHES`, `MUTATES`, `READS`, `EVENT_EMITS`, `EVENT_LISTENS`, `MIDDLEWARE_BEFORE`, `MIDDLEWARE_AFTER`
  - `SourceLocation` dataclass: `file_path`, `start_line`, `end_line`, `start_col`, `end_col`
  - Base node class with `node_type`, `label`, `source_location`, `metadata` (extensible dict)
  - Type-specific node subclasses: `FunctionNode(label, params, return_type, ...)`, `ClassNode(label, superclasses, ...)`, etc.

### 2. Code Emitter
- New file: `graphify/code_emitter.py`
  - `CodeEmitter` class with methods: `emit_call(caller, callee, confidence)`, `emit_import(file, target, strategy)`, `emit_extends(subclass, superclass)`, `emit_implements(class_, interface_)`, etc.
  - Confidence tiers: `1.0` for tree-sitter EXTRACTED, `0.7` for INFERRED (same-file), `0.5` for INFERRED (cross-file), `0.3` for AMBIGUOUS
  - `emit_batch(edges)` for bulk insertion

### 3. Typed Indexing Layer
- New file: `graphify/index.py`
  - `EdgeTypeIndex` — hash map `{EdgeType: [(source_id, target_id, data), ...]}` built at graph load. O(1) edge type filtering
  - `ConfidenceBitmapIndex` — three bit arrays (EXTRACTED, INFERRED, AMBIGUOUS) per edge type using Python `array('b')`. O(1) confidence-tier exclusion before traversal
  - `NodeLabelIndex` — prefix hash map for O(log n) node label lookup, replacing current `_score_nodes()` O(n) scan
  - `CompositeIndex` — unified index object that composes all three. `lookup(edge_type=CALLS, min_confidence=0.8, label_prefix="auth")` combines all filters
  - `build_indexes(graph) → CompositeIndex` — constructor called from `build.py`
  - `refresh_indexes(graph, added_edges, removed_edges)` — incremental update for post-commit reindexing

### 4. Enhanced Extraction
- Modified: `graphify/extract/__init__.py`
  - `extract(file_path, file_content)` now returns `(List[TypedNode], List[TypedEdge])` instead of generic concepts
  - Tree-sitter AST visitor produces type-specific nodes: function definitions → `FunctionNode`, class definitions → `ClassNode`, import statements → `ImportNode`, etc.
  - Edge emission uses `CodeEmitter` for proper confidence scores
  - Non-code files (docs, papers, images) continue producing generic `concept` nodes — backward compatible

### 5. Build Integration
- Modified: `graphify/build.py`
  - `build_from_json(graph_data, typed_schema=True, build_indexes=True)` — new parameters
  - After `nx.from_dict_of_dicts()`, calls `build_indexes(G)` and stores `G.graph["indexes"] = composite_index`
  - `G.graph["schema_version"] = 2` to mark typed schema
  - `G.graph["node_type_counts"] = {FunctionNode: 142, ClassNode: 23, ...}` stats

### 6. Migration Path
- New file: `graphify/migration.py`
  - `detect_schema_version(graph) → int` — reads `G.graph["schema_version"]`
  - `migrate_v1_to_v2(graph) → graph` — runs tree-sitter re-extraction on all source files in existing v1 graph to produce typed schema
  - `graphify migrate` CLI command — `graphify migrate path/to/graphify-out/` upgrades existing data

### 7. Tests
- New file: `tests/test_code_schema.py` — test every node type and edge type
- New file: `tests/test_code_emitter.py` — test confidence tiers, batch emission, edge deduplication
- New file: `tests/test_index.py` — test EdgeTypeIndex, ConfidenceBitmapIndex, NodeLabelIndex, CompositeIndex with graphs of 1k, 10k, 100k nodes
- New file: `tests/test_migration.py` — test v1→v2 migration on fixture repos
- Modified: `tests/test_extract.py` — verify typed output from fixture repos
- Modified: `tests/test_build.py` — verify indexed graph structure
