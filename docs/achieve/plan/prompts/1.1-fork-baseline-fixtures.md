# PR 1.1 — Fork + Baseline + Test Fixtures

**Epoch:** 1 (Foundation)
**Depends on:** None
**Estimated effort:** 0.5 weeks
**Impact:** Prerequisite

## Context

We are forking `safishamsi/graphify` to build an optimized, code-intelligence-enhanced version. This PR sets up the fork, benchmarks current performance, and creates the test fixture repos that all subsequent PRs will validate against.

## Deliverables

### 1. Fork Setup
- Fork `safishamsi/graphify` to `anomalyco/graphify` (or designated org)
- Clone, verify `uv sync && pytest tests/` passes clean
- Set up CI: lint (ruff), type check (mypy), unit tests, coverage report
- Add `90%` coverage threshold to CI config

### 2. Benchmark Suite
- New directory: `benchmarks/`
- New file: `benchmarks/bsbm.py` — generate BSBM-synthetic graphs at 1k, 10k, 50k, 100k, 500k node scales
- New file: `benchmarks/runner.py` — run all MCP tool calls (query_graph, get_node, get_neighbors, get_community, god_nodes, graph_stats, shortest_path) against each graph scale, record latency (p50, p95, p99) and memory
- New file: `benchmarks/baseline.json` — baseline results committed to repo

### 3. Language-Specific Test Fixture Repos
- New directory: `tests/fixtures/`
- New directory: `tests/fixtures/typescript/` — Express app with ~500 symbols: routes, controllers, services, middleware, models. Covers: class inheritance, interface implementation, middleware chains, route→handler→service call chains
- New directory: `tests/fixtures/python/` — Flask/FastAPI app with ~500 symbols. Covers: multiple inheritance (MRO), decorators, async handlers, dependency injection
- New directory: `tests/fixtures/go/` — Standard library-style package with ~500 symbols. Covers: interfaces, struct embedding, package-level functions, goroutines
- New directory: `tests/fixtures/java/` — Spring-style service with ~500 symbols. Covers: interfaces, abstract classes, annotations, method overrides, generics
- Each fixture includes a `graphify-out/` directory with expected `graph.json` (golden output) for schema validation

### 4. Baseline Recording
- Run `graphify build` on each fixture repo, record:
  - Build time
  - Node count, edge count (by type: concepts, calls, imports, etc.)
  - Current call resolution accuracy (manual count: how many cross-file calls are present vs missing)
- Commit `tests/fixtures/*/graphify-out/graph.json` as golden files

## Test Coverage Requirements

- **90%+** line coverage on all new files
- **100%** branch coverage on `benchmarks/runner.py` (it's a CLI tool — test every output mode)
- Fixture repos must have `pytest` test suites that exercise every symbol in the fixture

## Validation Criteria

- [ ] Fork exists, CI green, `pytest tests/` passes
- [ ] `python benchmarks/runner.py --scale all` completes without error
- [ ] `benchmarks/baseline.json` committed with latency/memory for all graph scales
- [ ] All 4 fixture repos have `pytest` suites that pass
- [ ] Golden `graph.json` files for all 4 fixtures committed
- [ ] Coverage report shows ≥90% on new code
