# PR 2.3 — Hybrid Search

**Epoch:** 2 (Core Intelligence)
**Depends on:** PR 1.2 (Code Schema + Typed Indexing)
**Estimated effort:** 2-3 weeks
**Impact:** High — ranked, semantic-aware codebase search

## Context

Graphify's current `query_graph` uses basic BFS/DFS traversal with naive term matching. This PR replaces it with BM25 keyword search + semantic vector search + Reciprocal Rank Fusion + process-grouped results. The default query mode becomes `"hybrid"` — BF/DFS remain available for backward compatibility.

Phase 6's approximate methods (node2vec, bloom filters, graph sampling) are evaluated here. Text embeddings (semantic) are the priority; structural embeddings (node2vec) are deferred to a future `graphify/search/structural/` module with shared infrastructure.

## Deliverables

### 1. Search Orchestrator
- New file: `graphify/search/__init__.py`
  - `SearchOrchestrator` class
  - `hybrid_search(query_text, graph, options) → GroupedSearchResults`
  - Options: `bm25_weight=0.4`, `semantic_weight=0.4`, `process_weight=0.2`, `limit=20`, `min_confidence=0.5`, `scope=ALL | BY_COMMUNITY | BY_PROCESS`
  - Runs BM25 + semantic in parallel, fuses via RRF, groups by process, returns

### 2. BM25 Keyword Search
- New file: `graphify/search/bm25.py`
  - `BM25Index` class — built at graph load time
  - Indexed fields per node: `label`, `file_path`, `docstring` (extracted from AST comments), `type_name` (Function, Class, etc.)
  - Tokenization: lowercased, stripped punctuation, min length 2
  - `search(query, top_k=100) → List[(node_id, score)]`
  - `incremental_update(added_nodes, removed_nodes)` — only re-index changed nodes
  - Stored in `G.graph["bm25_index"]` for fast access

### 3. Semantic Vector Search
- New file: `graphify/search/embeddings.py`
  - `EmbeddingIndex` class
  - Embedding model: `all-MiniLM-L6-v2` (384D, small, fast, good code similarity) — configurable
  - `embed_node(node)` — concatenate label + docstring + type information, generate embedding
  - Only embed code nodes (Function, Class, Method, Interface) — skip generic concept nodes
  - `search(query_text, top_k=100) → List[(node_id, cosine_similarity)]`
  - `incremental_update(graph, changed_node_ids)` — SHA1-based staleness detection per node
  - Stored in `graphify-out/embeddings/` with sharding: >50k symbols splits into 10k-node shards
  - `search_shard(query_embedding, shard_path) → List[(node_id, score)]` — binary vector search per shard
  - CLI flag: `graphify build --embeddings` (opt-in due to dependency weight)
  - Optional dependency: `[embeddings]` extra in pyproject.toml (sentence-transformers)

### 4. Reciprocal Rank Fusion
- New file: `graphify/search/fusion.py`
  - `reciprocal_rank_fusion(bm25_results, semantic_results, graph_results, k=60) → List[(node_id, fused_score)]`
  - Formula: `RRF_score(d) = Σ(1 / (k + rank_i(d)))` for each ranker `i`
  - `k=60` standard value, robust to arbitrary score scales
  - `merge_ranked_lists(lists, k=60)` — merges N ranked results lists
  - Handles ties: first-seen wins for deterministic output

### 5. Process-Grouped Results
- New file: `graphify/search/grouping.py`
  - `group_by_process(search_results, graph) → GroupedSearchResults`
  - Groups results by which process they belong to (from PR 2.2's `STEP_IN_PROCESS` edges)
  - Each group: `process_name`, `summary_text`, `symbols: [{name, type, file, score, step_index}]`, `priority_score`
  - Priority score: density of matching symbols in the process (matches / total steps)
  - Ungrouped results (symbols not in any process) returned as `orphaned: [...]`
  - Cross-community results flagged: symbol belongs to a different community than the top result

### 6. Modified: `serve.py` Default Search
- Replace `_bfs()` as default `query_graph` implementation
- New: `_hybrid_query(query_text, graph, options)` — calls `SearchOrchestrator.hybrid_search()`
- `query_graph` default mode changed from `"bfs"` to `"hybrid"`
- Old modes: `mode="bfs"`, `mode="dfs"` still available
- New MCP tool `query` (hybrid default):
  ```json
  query({query: "authentication middleware", limit: 10})
    → processes: [{summary, priority, symbol_count, process_type, step_count, symbols: [
        {name, type, file, line, confidence, process_context}
      ]}]
    → references: [{name, type, file, process_context, score}]
    → orphaned: [{name, type, file, score}]
    → total_results: N
    → search_metadata: {bm25_hits, semantic_hits, fusion_time_ms}
  ```

### 7. Future: Structural Embeddings (Deferred)
- File: `graphify/search/structural.py` (NOT in this PR, placeholder only)
  - `StructuralEmbeddingIndex` — node2vec/PyKEEN for graph-structure similarity
  - `search(node_id) → List[(node_id, structural_similarity)]`
  - Fused via same RRF pipeline with text embeddings
- Reason for deferral: text embeddings have higher ROI; structural embeddings require model training/tuning per graph
