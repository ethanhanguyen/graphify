# PR 3.2 — Multi-Repo Groups

**Epoch:** 3 (Integration & Scale)
**Depends on:** PR 1.2 (Code Schema + Typed Indexing)
**Estimated effort:** 2-3 weeks
**Impact:** Medium (Enterprise tier)

## Context

For monorepo and microservice architectures, users need a unified knowledge graph across multiple repositories. This PR adds repository groups with cross-repo contract bridging, lazy connection pooling, and group-aware MCP tools. It is additive — existing single-repo workflows continue unchanged.

## Deliverables

### 1. Global Registry
- New file: `graphify/registry.py`
  - `Registry` class backed by `~/.graphify/registry.json`
  - `register(repo_path, meta=None) → repo_id` — register a new repo
  - `unregister(repo_id)` — remove repo from registry
  - `list_repos() → List[RepoEntry]` — all registered repos
  - `get(repo_id) → RepoEntry` — single repo metadata
  - `update(repo_id, last_indexed=None, head_commit=None, status=None)` — update metadata
  - `RepoEntry` dataclass: `id`, `path`, `name`, `last_indexed`, `head_commit`, `language`, `node_count`, `edge_count`, `groups: List[str]`
  - Atomic writes: write to temp file, rename (prevents corruption)

### 2. Lazy Connection Pool
- New file: `graphify/lazy_pool.py`
  - `GraphPool` class
  - `get_graph(repo_id) → NetworkXGraph` — open on first query, keep in pool
  - `evict(repo_id)` — close graph, free memory
  - `evict_idle(max_idle_seconds=300)` — evict graphs inactive for >5 minutes
  - `max_pool_size = 5` — evict LRU when pool exceeds limit
  - `pool_stats() → {open_count, idle_count, memory_mb}`
  - Thread-safe: `threading.RLock()` guards pool dict
  - Lazy index loading: indexes (from PR 1.2) loaded on first query, not graph open

### 3. Repository Groups
- New file: `graphify/groups.py`
  - `GroupManager` class backed by `~/.graphify/groups.json`
  - `group_create(name, description="") → Group`
  - `group_add(group_name, repo_path_or_id, hierarchy_path="")` — add repo to group at path (for nested groups like `backend/auth` vs `backend/payment`)
  - `group_remove(group_name, repo_path_or_id)`
  - `group_list() → List[Group]`
  - `group_sync(name, full=False)` — sync group:
    1. For each repo in group, check staleness (`head_commit != registry.last_indexed`)
    2. If stale, run `graphify build` on that repo
    3. Extract contracts: shared interfaces, API routes, type exports across repos
    4. Build bridge edges using `ContractBridge`
  - `group_query(name, query_text, options) → GroupQueryResults`
    - Fan-out: run hybrid search (PR 2.3) on each repo in group
    - Merge results via Reciprocal Rank Fusion
    - Return per-repo results + merged results
  - `group_status(name) → GroupStatus`
    - `repos: [{repo, last_indexed, head_commit, stale: true/false}]`
    - `contracts_count`, `bridges_count`, `last_synced`
  - `Group` dataclass: `name`, `description`, `members: List[GroupMember]`, `created_at`, `last_synced`
  - `GroupMember` dataclass: `repo_id`, `path`, `hierarchy_path`

### 4. Contract Bridge
- New file: `graphify/contract_bridge.py`
  - `ContractBridge` class
  - `extract_contracts(graph, language) → List[Contract]`
    - `Contract` = shared interface or API surface: type name, language, file, exported symbols, signature hashes
  - `build_cross_repo_bridges(contracts_a, contracts_b) → List[BridgeEdge]`
    - Matches contracts by: identical type name + matching signature hash (parameter types, return type)
    - Confidence scores: exact match = 1.0, name-only = 0.5, partial signature match = 0.7
    - `BridgeEdge(email)` = `{source_repo, target_repo, source_symbol, target_symbol, relation, confidence}`
  - `detect_shared_apis(graph) → List[APIEndpoint]` — API route → handler → downstream consumers
  - `map_consumers(graph, api_endpoint) → List[Consumer]` — which symbols depend on this API

### 5. Group-Aware MCP Tools
- New in `serve.py` (group mode):
  ```
  group_list() → [{name, member_count, repo_paths, last_synced, status}]
  
  group_sync({name: "my-services"}) → {contracts_found, bridges_created, repos_synced, staleness: [...]}
  
  group_contracts({name: "my-services"}) → {providers: [...], consumers: [...], cross_links: [...]}
  
  group_query({group: "my-services", query: "auth middleware", limit: 20}) → {
    repo_results: {repo_name: [...results]},
    merged: [...RRF merged results],
    total_results: N
  }
  
  group_status({name: "my-services"}) → {
    repos: [{repo, last_indexed, head_commit, stale, node_count, edge_count}],
    total_nodes, total_edges, contracts_count, bridges_count
  }
  ```
- Updated: `context`, `impact`, `query` tools accept optional `repo` parameter
  - When omitted and only 1 repo indexed, auto-select
  - When `repo="@groupName"`, fan out across group members, merge via RRF

### 6. CLI Commands
- New subcommand: `graphify group`
  ```
  graphify group create <name> [--description]
  graphify group add <name> <repo-path> [--path <hierarchy>]
  graphify group remove <name> <repo-path>
  graphify group list
  graphify group sync <name> [--full]
  graphify group status <name>
  graphify group query <name> <query-text>
  ```
- New subcommand: `graphify register`
  ```
  graphify register [--name <name>] [--group <group>]
  graphify unregister <repo-id>
  graphify registry list
  ```
